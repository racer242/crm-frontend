Ознакомься с документацией в docs - здесь описаны принципы устройства системы, описаны все компоненты системы
Ознакомься с документами .prompts\docs\api - здесь техническое задание для разработчика внешнего API, предоставляющего доступ к данных системы. API реализовано и мы к нему будем подключаться.
Ознакомься с конфигурацией config\crm-config.json - это скелет нашей CRM
Ознакомься с конфигурацией config\pages\users.json - это готовая страница для вывода списка участников
Ознакомься с роутами в config\system\api-routes.json - это список всех текущих роутов для подключения к внешнему API
Ознакомься с адаптерами:
config\adapters\users-params.js - адаптирует параметры запроса от фронтенда CRM в параметры запроса к внешнему API - применительно к запросу users
config\adapters\users-2-tabview.js - адаптирует ответ внешнеого API в ответ для фронтенда CRM.

На основании представленных материалов и готовой страницы users (эта страница (и принадлежещие ей артифакты) готова, ее нельзя трогать и менять, на нее надо ориентироваться)
Тебе необходимо добавить в CRM аналогично работающие страницы для разделов:

- Акты
- Призы
- Чеки
- Коды
- Коды Честный Знак

Все эти разделы выдают списки данных в виде таблиц.
Если одноименная страница уже существует - старую можно удалить (заменить на новую)
