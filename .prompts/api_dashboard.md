# API для дашборда CRM

## Анализ существующих роутов и данных для дашборда

---

## 1. Анализ текущих роутов (config/system/api-routes.json)

### Существующие роуты, пригодные для дашборда:

| Роут              | Метод | Что даёт                                                                                                   | Пригодность для дашборда                                                |
| ----------------- | ----- | ---------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------- |
| `stats/review`    | GET   | Быстрая статистика — обзорные таблицы (активные пользователи, свод по призам, свод по кодам, Честный Знак) | ✅ Частично — даёт агрегированные данные, но без специфических разрезов |
| `stats/weeks`     | GET   | Список недель для фильтрации                                                                               | ✅ Полезен для селектора периода                                        |
| `stats/analytics` | GET   | Древовидное меню ссылок на скачивание xlsx-отчётов                                                         | ✅ Полезен для выгрузок                                                 |
| `prizes`          | GET   | Список всех призов с пагинацией                                                                            | ✅ Даёт список призов, но без агрегации по статусам                     |
| `users`           | GET   | Список участников                                                                                          | ⚠️ Слишком детально, нужна агрегация                                    |
| `receipts`        | GET   | Список чеков                                                                                               | ⚠️ Слишком детально                                                     |
| `codes`           | GET   | Список кодов                                                                                               | ⚠️ Слишком детально                                                     |
| `gtins`           | GET   | Список кодов ЧЗ                                                                                            | ⚠️ Слишком детально                                                     |

### Чего не хватает для полноценного дашборда:

1. **Единого агрегированного endpoint'а** со сводными метриками по кампании
2. **Специфических срезов**, которые используют менеджеры (см. данные проекта ниже)
3. **Статистики по промокодам** (выдача, остатки)
4. **Воронки конверсии** (регистрация → коды → чеки → призы)
5. **Причин отклонения** кодов и чеков
6. **Еженедельной статистики** по шатровым играм, каруселям и т.д.
7. **Топ-городов** и **топ-вкусов**
8. **Динамики по дням** (чеки, регистрации)


**Возможные столбцы:** `Приз | Всего | Выиграно | Заказано | Получено | Отклонено | Ожидает`
(состав столбцов зависит от бизнес-логики конкретной кампании)

**Визуализация:**

- Круговая диаграмма (распределение по статусам для выбранного приза)
- Составная столбчатая диаграмма (stacked bar — по каждому призу сегменты статусов)
- Тепловая карта (heatmap) приз×статус

**Варианты группировки:**

1. **Плоская таблица** — приз + каждый статус в отдельном столбце
2. **Сгруппированная по типу** — электронные / физические / баллы
3. **Сгруппированная по статусу** — раскрывающиеся группы (заказанные / выигранные / полученные)

### 2.4. Промокоды — сводная таблица

Единая таблица промокодов с колонками:

```
Приз                            Код приза       Всего в пуле    К выдаче    Остаток    Выдано %
Промокод Додо Пицца             DODO_PROMO         20 000        13 979      6 021     69.9%
Промокод Додо Пицца             DODO_PROMO_2       20 000             0     20 028    100.1%*
Промокод Крошка Картошка        KROSHKA            20 000        15 398      4 602     77.0%
Промокод Лукойл                 LUKOIL             20 000        13 687      6 314     68.5%
Промокод Лукойл                 LUKOIL_2           20 000         1 898     18 102     90.5%
Билет на Фестиваль Grounded     GROUNDED              200             0        200    100.0%
Промокод Киномакс               KINOMAX            10 000         6 755      3 245     67.6%
Промокод Киномакс               KINOMAX_2          10 000         1 133      8 867     88.7%
100 баллов Яндекс Плюс          YANDEX_PLUS100     12 000         3 562      8 438     70.3%
500 баллов Яндекс плюс          YANDEX500           1 000         1 000          0      0.0%
```

> _Примечание: Выдано % может превышать 100%, если в пул добавлялись доп. коды или есть ручная выдача сверх пула._

**Формулы:**

- `К выдаче` = `Остаток` (ещё не использованные коды из пула)
- `Остаток` = `Всего в пуле - Выдано`
- `Выдано %` = `Выдано / Всего в пуле * 100`

**Визуализация:**

- Прогресс-бары по каждому промокоду (оставшаяся часть vs выданная)
- Столбчатая диаграмма (пул vs остаток)
- Индикаторы "low stock" для промокодов с остатком < 10%

## 4. Типы визуализаций для данных дашборда

### 4.1. Плитки (KPI Cards)

- **Для:** summary метрик, weekly-report
- **Содержание:** Иконка + число + подпись + sparkline-график тренда + % изменения
- **Примеры:** Всего регистраций (63 097), Всего кодов (211к), Принято чеков (38 101)

### 4.2. Воронка (Funnel Chart)

- **Для:** `/dashboard/funnel`
- **Отображение:** 5-6 этапов конверсии с процентом перехода
- **Кастомный** цветовой градиент (зелёный → жёлтый → красный)

### 4.3. Горизонтальные столбцы (Horizontal Bar)

- **Для:** Распределение призов, Топ-10 городов, Топ-5 вкусов
- **Сортировка:** по убыванию
- **Лимит:** Top N (настраивается)

### 4.4. Линейные графики (Line Chart)

- **Для:** Динамика по дням (регистрации, коды, чеки), тренды
- **Multi-line:** несколько метрик на одном графике
- **Зона:** fill между линиями для наглядности

### 4.5. Составные столбцы (Stacked Bar)

- **Для:** Выборка по призам (приз × статусы), загрузка чеков по дням
- **Цвета:** зелёный (получено), синий (заказано), жёлтый (ожидает), красный (отклонено)

### 4.6. Круговая / Кольцевая (Pie / Donut)

- **Для:** Причины отклонения, источники загрузки, распределение типов призов
- **Лучше Donut** — можно в центр поместить общее число

### 4.7. Прогресс-бары (Progress Bars)

- **Для:** Промокоды (процент выдачи), остатки пула
- **Цвет:** зелёный > 50%, жёлтый 25-50%, красный < 25%

### 4.8. Таблицы (DataTable)

- **Для:** Все призы, Выборка по призам, Промокоды, Отклонённые коды
- **Возможности:** сортировка по столбцам, поиск, экспорт

### 4.9. Sparkline-графики в плитках

- **Для:** Мини-график тренда внутри каждой KPI-плитки
- **Показывает:** направление (вверх/вниз) и амплитуду изменения

### 4.10. Карта (Geo Chart)

- **Для:** Топ-городов — можно отобразить на карте России с пузырьками
- **Размер пузырька:** пропорционален числу регистраций

## 7. Рекомендации по архитектуре дашборда

### 7.1. Формат пути

Все дашборд-эндпоинты следуют формату ТЗ:

```
/api/v1/{campaign_id}/dashboard/{resource}
```

### 7.2. Кэширование

Дашборд-данные рекомендуется кэшировать (TTL 5-10 минут), так как они:

- Тяжеловесные для подсчёта (агрегации по большим таблицам)
- Не требуют секундной актуальности

### 7.3. Пагинация и лимиты

Для таблиц (призы, промокоды и т.д.) использовать стандартные параметры `first`, `limit`.
Для чартов и плиток лимиты фиксированные (top 10, top 5).

### 7.4. Приоритет реализации

**Высокий приоритет (MVP дашборда):**

- `/dashboard/summary` — сводка метрик
- `/dashboard/prizes` — распределение призов
- `/dashboard/prizes-breakdown` — сводная по статусам
- `/dashboard/promocodes` — промокоды
- `/dashboard/funnel` — воронка
- `/stats/review` (доработка) — быстрая статистика

**Средний приоритет:**

- `/dashboard/rejected-codes` — отклонённые коды
- `/dashboard/registrations-by-day` — динамика
- `/dashboard/top-cities` — города
- `/dashboard/trends` — тренды для sparkline

**Низкий приоритет:**

- `/dashboard/weekly-report` — еженедельный отчёт
- `/dashboard/receipt-sources` — источники
- `/dashboard/referral` — рефералы
- `/dashboard/receipt-summary` — сумма чеков
- `/dashboard/top-flavors` — вкусы
- `/dashboard/code-sources` — источники кодов
- `/dashboard/receipts-by-day` — чеки по дням
- `/dashboard/rejected-receipts` — отклонённые чеки
- `/dashboard/points` — баллы
