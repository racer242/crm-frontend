.prompts\docs\api\Данные участника.md

Конфигурация страницы вывода данных пользователя здесь: config\pages\user.json

На нее осуществляется переход в виде http://host/users/23747
где 23747 - это id пользователя.
В рамках конфигурации страницы, его можно получить через макрос location (изучи как).

Сейчас страница содержит какие-то фейковые секции.
Тебе нужно создать страницу так, чтобы она выводила все доступные данные участника согласно эндпойнту.

Запрос должен осуществляться через свойство dataFeed (серверная загрузка данных перед серверным рендером, см docs\data-feed-reference.md), аналогично как на странице config\pages\users.json загружается список пользователей.

Для получения данных используется роут "users/[id]". Возможно нужен адаптер для конвертации полученных данных (response) пользователя (docs\data-adapter-reference.md).

Визуализация должна быть построена на компонентах системы docs\components-reference.md
Скомпонуй страницу, типичную для просмотра пользователя, с использованием известных данных эндпойнта.
