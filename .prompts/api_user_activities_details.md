# Создай страницы подробного просмотра записи:

config\pages\user-act.json - страница просмотра акта, данные (api/acts/[id]):

- ID, Имя фамилия участника - есть ссылка на переход к участнику
- Email участника - есть ссылка написать участнику (ссылка открытие почтового клиента по умолчанию)
- Дата выдачи акта
- Приз
- Статус
- Бланк документа - ссылка Скачать
- Фото подписанного скана - ссылка Скачать
- Редактировать - Кнопка перехода на роут редактированию
- Удалить - Кнопка удаления, с вызовом подтверждения

config\pages\user-prize.json - страница просмотра приза, данные (api/prizes/[id]):

- ID, Имя фамилия участника - есть ссылка на переход к участнику
- Email участника - есть ссылка написать участнику (ссылка открытие почтового клиента по умолчанию)
- Дата выигрыша
- Приз - название приза
- Торговая сеть - название торговой сети
- Статус - бейдж
- Комментарий - текст
- Личные документы - переход на роут страницы config\pages\user-docs.json
- Акт - переход на роут страницы config\pages\user-act.json
- Добавить акт - Кнопка перехода на роут - добавление акта к призу
- Редактировать - Кнопка перехода на роут редактирования приза
- Удалить - Кнопка удаления, с вызовом подтверждения

config\pages\user-receipt.json - страница просмотра чека, данные (api/receipts/[id]):

- ID, Имя фамилия участника - есть ссылка на переход к участнику
- Email участника - есть ссылка написать участнику (ссылка открытие почтового клиента по умолчанию)
- Дата регистрации чека
- Дата покупки
- Торговая сеть
- Сумма акционных продуктов
- Сумма общая
- Фискальные данные
- Статус чека
- Статус модерации
- Комментарий - комментарий последней модерации
- Список продуктов - Таблица (данные из api/receipts/[id]/products) столбцы:
  - Продукт
  - Количество
  - Сумма
  - Свойство 1
  - Свойство 2
  - Акционный
- Модерировать - Кнопка перехода на роут редактирования чека
- Удалить - Кнопка удаления, с вызовом подтверждения

config\pages\user-qtin.json - страница просмотра кода Честный знак (ЧЗ), данные (api/gtins/[id]):

# Создай страницы редактирования/добавления записей:

- Изменить статус - Выбор из списка
- Добавить - кнопка добавления акта (для страницы добавить акт)

- Торговая сеть - Выбор из списка
- Статус - Выбор из списка
- Комментарий - Ввод строки

- Добавить - кнопка добавления приза (для страницы добавить приз)

# Прочие инструкции

Вся конфигурация аналогична той, что на страницах:

- config\pages\user-edit.json
- config\pages\user-docs-edit.json

На эти страницы будут осуществляется переходы в виде http://host/users/23747/...
где 23747 - это id пользователя.

Запросы осуществляеются через свойство dataFeed (серверная загрузка данных перед серверным рендером, см docs\data-feed-reference.md). Табличные данные поставляются с поисковыми строками и фильтрами, через обобщенные настройки команд в shortcuts (см docs\linkage-reference.md, docs\macros-reference.md).

Для получения данных используются соответствующие роуты.
Для всех роутов надо предусмотреть адаптеры (см. docs\data-adapter-reference.md), по аналогии с:
config\adapters\user.request.js - подготовка параметров
config\adapters\user.js - подготовка ответа API

Визуализация должна быть построена на компонентах системы docs\components-reference.md

Скомпонуй страницы, типичную для просмотра соответствющих данных, с использованием известных данных эндпойнта.
