Я хочу сделать возможность переключаться между разными кампаниями.
Принцип - в конфигурации есть список API кампаний в виде:

camps:[
{"id":7384, "name":"Добрый Кола 2026", "api_url":"https://dev.ssd26.srv08.ru/api/v2/7384"},
...
]

этот список вынесен в файл config\system\camps.json и подключен в основной конфигурации config\crm-config.json

Пользователь может выбрать, какой кампанией он пользуется.
Это делается в дашборд-меню, под блоком авторизации пользователя - текущая кампания выводится, а все остальные - раскрывающимся списком (по аналогии с меню пользователя)

При этом, для меню из camps выдается только связка "id"->"title" (адреса api не должны попадать на фронтенд)

ID текущей кампании, который выбрал пользователь сохраняется в куки (или localstorage) и при отправке в next-api-роут любого запроса api передается.

next-api-роут принимая запрос от пользователя получает id кампании, выбирает из списка camps нужный url и делает туда обращение.

в config\system\api-routes.json теперь не нужны макросы {$env.API_CAMP_URL_0}. Роутер сам будет добавлять нужный адрес к адресу api из api-routes. Также не нужны API_CAMP_URL_0 в .env.\*
