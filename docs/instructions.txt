Я хочу сделать универсальную CRM-платформу. 
Примерная концепция визуализации и структуры данных в файлах docs/analytics/Визуализация.pdf и docs/analytics/Данные.pdf.

Концепция представляет собой архитектуру, которая лежит в основе многих современных low-code/no-code платформ. 
Это так называемый "Metadata-Driven UI" (интерфейс, управляемый метаданными), где структура, внешний вид и поведение приложения 
описываются в виде данных (в вашем случае — JSON), а не жестко кодируются.

Система будет состоять из трех основных частей:
1.  "Чертеж" (The Blueprint) — JSON-конфигурация, описывающая всю структуру CRM.
2.  "Движок" (The Engine) — Frontend-приложение на Next.js, которое читает "Чертеж" и на его основе строит интерфейс, управляет состоянием и выполняет команды.
3.  "Источник данных и логики" (The Backend) — API, которое предоставляет данные для CRM и принимает запросы на управление данными.

Система строится на иерархии элементов: клиент (nextjs app) > страница > секция > блок > компоненты (из которых строятся блоки) .
Тип блока определяет набор компонентов, их виды, расположение.
Каждый элемент обладает идентификатором, своим состоянием, специфичными параметрами.
Интерактивность обеспечивается интерактивными компонентами, которые могут выполнять те или иные команды. Данные элементов доступны по единой системе адресации основанной на путях, построенных на ID элементов.
Команды могут влиять на состояния элементов, делать запросы - после выполнения происходит рендер задействованных элементов. Также они переходить по страницам, секциям, ссылкам и т.п. 
Команды могут выполняться при старте страницы, при воздействии на интерактивные элементы, по таймеру, при каких-либо условиях.

CRM изначально конфигурируется в виде JSON-файла определенной структурой.
Динамические данные (получаемые по запросам команд - например данные таблицы) формируются по единым правилам, определенным для каждого из компонентов. 
Динамические данные могут модифицировать текущую структуру страницы, например, показывать новые элементы, скрывать или показывать элементы, 
отображать в других компонентах те или иные данные.

Предусматривается слой, адаптирующий ответы API к формату Платформы. 
Дело в том, что API не разрабатывается нами или под нас. Не API подстраивается под Платформу, а 
платформа должна понимать API. 

Технологический стек - NodeJS, NextJS, Vite, PrimeReact, TS, REST, Docker

Прилагаются: 
- функциональные требования для разработки платформы.
- описание структуры данных, элемены, параметры элементов, набор компонентов.
- описание команд. 
- описание принципов взаимодействия элементов.
- последовательность разработки, правила и требования к коду.
Эти документы разположены в папке docs/inference
