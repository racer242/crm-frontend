1. Вводные

Проект представляет универсальную CRM-платформу. 
Технологический стек - NodeJS, NextJS, PrimeReact, TS, REST, Docker

Система строится на иерархии элементов: клиент (nextjs app) > страница > секция > блок > компоненты (из которых строятся блоки).
Тип блока определяет набор компонентов, их виды, расположение.
Каждый элемент обладает идентификатором, своим состоянием, специфичными параметрами.

Интерактивность обеспечивается интерактивными компонентами, которые могут выполнять те или иные команды. 
Данные элементов доступны по единой системе адресации основанной на путях, построенных на ID элементов.
Команды могут влиять на состояния элементов, делать запросы - после выполнения происходит рендер задействованных элементов. 
Также они переходить по страницам, секциям, ссылкам и т.п. 
Команды могут выполняться при старте страницы, при воздействии на интерактивные элементы, по таймеру, при каких-либо условиях.

CRM изначально конфигурируется в виде JSON-файла с определенной структурой.
Файл конфигурации находится в корне проекта (вне Docker-контейнера, переменная окружения CONFIG_PATH). 
Файл не импортируется в проект, а загружается на бэкенде при инициализации серверной части приложения CRM.
Файл может быть составным - используется стандарт $ref. 

Динамические данные (получаемые по запросам команд - например данные таблицы) формируются по единым правилам,
определенным для каждого из компонентов. 
Динамические данные могут модифицировать текущую структуру страницы, например, показывать новые элементы, 
скрывать или показывать элементы, отображать в других компонентах те или иные данные.

Предусматривается слой, адаптирующий взаимодействие с внешним API к формату Платформы. Внешний API подстраивается под Платформу, а 
платформа должна взаимодействовать с API.

Апп разрабатывается ИИ-агентом на платформе VS-Code.
