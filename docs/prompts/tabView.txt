Посмотри как работает компонент TabView (src\engine\components\ContainerComponents.tsx)
1. Зашита хардкодом таб-панель-таблица. 
2. Поле content является невнятной реализацией - видимо рассчитаной только на вывод текста.
Это ужасно.

Что я хочу:
1. Представь что есть секция с набором блоков, но они не располагаются один под другим или слева направо. 
Блоки переключаются табами. Это и есть принцип работы TabView.

2. Как это будет выглядеть в конфигурации.
...Блок
"components": [
  {
    "id": "statsTabView",
    "type": "component",
    "componentType": "TabView",
    "props": {
    }
    "tabs": [
      {
        Это Блок с враппером TabPanel!
        "state": {},
        "props": {
          "header": "Покупатели"
        }
        "components": [
          ...Конпоненты, выводимые в блоке-таб-панели
        ]
      }
    ]  
  }
]

3. Таким образом, компонент TabView становится "переключателем" блоков, в которых в свою очередь выводятся компоненты
4. Каждый блок в TabView -  это эквивалент блокв, у которого указан враппер: 
"wrapper": {
  "component": "TabPanel",
  "props": {
    "header": "Text"
  }
} 
4. Аналогично меняем Accordion - здесь враппер - AccordionTab
5. После обновления функционала, необходимо поправить и файлы конфигурации, где встречаются данные компоненты
6. Зачем нужен "blockType": "..." ? Я не увидел, чтобы он где-либо применяется. Надо от него избавиться.
