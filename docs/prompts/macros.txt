Надо вывести на более глобальный уровень системы управления данными приложения. 
Необходимо их систематизировать, сделать подходы более централизованными и лаконичными в применении.
Требуется унифицированный лаконичный вызов функций в коде. Каждая система может быть реализована как класс-синглтон.
например, value = Linkage.resolve(page.value) или value = Macro.apply(page.value)
Данные, поставляемые в рабочие функции могут быть из разных источников. Все источники передаются в класс-синглтон 
заранее, при инициализации, а не при каждом вызове - это сделает код лаконичным.


Приложение реализует две системы:
1. Система линковки. Предназначена для связи свойств элементов с состояниями state. Отслеживает изменение state и обновляет прилинкованные элементы. Важно - линковка нужна для автоматического рендера данных.
2. Система макросов. Предназначена для подстановки данных в параметры команд при их срабатывании, в парамтры конфигурации, при доступе к ним, возможны и иные применения. Важно - макросы нужны для добычи/вычисления данных и их подстановки в конкретных вызовах.


Система линковки

1. Линковка полностью копирует принцип React, когда есть state, и компоненты в своих параметрах используют совйтсва state, а при изменении state, они производят рендер.
2. Каждый элемент имеет свой state и линковка может осуществляться от любого state к любому свойству любого элемента. Например, это какой-либо из params компонента - params.showIcon, params.placeholder, или свойство вложенного объекта - params.style.backgroundColor
3. Изменение определенного state влечет рендер только линкованных с ним элементов.
4. Изменение state возможно при помощи команд setProperty, setState, sendRequest и т.п. В любой из этих команд есть параметр target, в котором указывается путь к state или его полю.
5. Если в state изменяется только одно свойство, обновляются все связанные со state элементы. Если есть возможность - только связанные с этим свойством элементы.
6. Линковка возможна только в рамках страницы.
7. Формат линковки: "value"="@ELEMENT_ID.state.PATH.TO.FIELD",  где:
"@" - обозачает, что параметр линкуется.
"ELEMENT_ID" - элемент, обладающий state, к которому линкуется. Если "ELEMENT_ID." отсутствует, это означает, что state используется корневой - принадлежит элементу типа "page" - странице.
"state" - означает, что линкуется state. Возможно в дальнейшем будут предусмотрены другие источники линковки.
"PATH.TO.FIELD" - путь к свойству state. Это может просто переменная, а может и свойство объекта, находящегося в state.



Система макросов

1. Макросы предназначены для формирования данных в командах, конфигурациях, возможны иные применения. 
Например, при нажатии кнопки, осуществляется отправка данных формы, при этом в объекте отправляемых данных, каждое поле имеет заначения-макросы, содержащие ссылки на значения в state блока формы или state других элементов.
Кроме того, макрос может содержаться в url запроса - такие данные могут храниться в конфигурации приложения.
2. В одном значении может быть несколько макросов.
3. После применения макроса, в значении может возникнуть наличие нового макроса (в подставленном вычисленном значении макроса были еще макросы)
4. Подстановка макросов всегда должна быть динамической - при каждом доступе к значению, поскольку подставляемые данные изменны. 
5. Необходимо учитывать, что некоторые макросы применяются только на клиентской, и есть те, которые могут использоваться и на сервере и на клиенте. 
Например, макросы, ссылающиеся на state не могут использоваться на сервере. Макросы конфигурации приложения могут использоваться везде.
7. Виды макросов.
7.1. Макросы-ссылки. Ссылаются на свойства state для получения их значений. {$ELEMENT_ID.state.PATH.TO.FIELD}
7.2. Макросы конфигурации. Ссылаются на данные в разделе конфигурации config. {$config.PROP}
7.3. Макросы URL. Ссылаются на параметры адресной строки. 
{$location.PROP} - предусматривает все возможные значения location: href, protocol, host, hostname, port, pathname, search, hash, origin
{$location.query.PARAM} - подставляет значение параметра url-строки "?name1=value1&name2=value2", где PARAM - name1 или name2, а подставится value1 или value2.
{$location.slug.N} - подставляет значение сегмента "/slug0/slug1/slug2", где N - 0, 1 или 2, а подставится slug0, slug1 или slug2.
7.4. Макросы времени и даты - подставляет значения даты.
{$now.iso} - подставляет текущую дату в формате ISO: "2024-05-20T10:00:00.000Z"
{$now.timestamp} - подставляет числовое значение текущего вермени 1716200000000
{$now.DD.MM.YY} - подставляет значение текущего вермени "22.04.26"
{$now.DD.MM.YYYY} - подставляет значение текущего вермени "22.04.2026"
{$now.DD.MM.YYYY HH:mm} - подставляет значение текущего вермени "22.04.2026 01:49"
{$now.DD.MM.YYYY HH:mm:ss} - подставляет значение текущего вермени "22.04.2026 01:49:15"
{$now.DD.MM.YYYY HH:mm:ss:SSS} - подставляет значение текущего вермени "22.04.2026 01:49:15:829"
7.5. Макросы сессии. {$session.PROP}
7.6. Макросы localStorage. {$localStorage.PROP}
7.7. Макросы Куки. {$cookie.PROP}
7.8. Макросы Window. {$window.PROP}
7.9. Макросы user. {$user.PROP}
7.10. Макросы auth. {$auth.PROP}
8. Типы макросов
Макросы всегда представлены как строки, поскольку JSON иначе не позволит.
Однако, если макрос при подстановке имеет иной тип, он должен подставиться именно в том типе, которые был возвращен.
Например:
Параметр с макросом: "param1":"{$form.state.selectedItems}". Макрос подставляет в свойство объекта массив param1: ["value0","value2","value5"].
Параметр с макросом: "param1":"{$form.state.checked}". Макрос подставляет в свойство объекта Boolean param1: true.
Параметр с макросом, передающим весь state: "data":"{$form.state}" - подставит весь объект state в свойство data объекта.

Если в строке несколько макросов, необходимо сначала их получить, а потом собирать. 
Главное правило сборки: тип определяется первым макросом. 
Например (имеется в виду последовательность макросов, приводятся сразу полученные типы):
Строка1Строка2Строка3 - строки сложатся. Например "{$config.baseURL}/{$state.page}/?prop={$state.param1}" подставит "https://localhost:3000/statistics/?prop=foo".
Объект1Объект2Объект3 - Тип подстановки - Объект. Результат - Объект1, на который наложится с замещением Объект2 и затем на полученное - Объект3. Например "{$state}{$form.state}{$section.state}" даст объединенный объект.
Объект1СтрокаBooleanNumber - В Объект1 добавятся поля prop1:"Пример", prop2:true, prop3: 3123.
СтрокаBooleanNumber - получится "Примерtrue3123".
BooleanСтрокаNumber - все значения будут приведены к Boolean и сложены.
NumberСтрокаBoolean - все значения будут приведены к Number и сложены.
СтрокаОбъектОбъект - все объекты будут сериализованы.
Если в строке помимо макроса/макросов есть хотя бы один строковый символ, вся последовательность приводится к строковому типу.
