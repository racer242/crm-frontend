Работа с внешними API.

Может осуществляться как на сервере так и на клиенте.

#Формат запроса.

Описание запроса имеет следующий фармат:
{
  "url": "URL/TO/API/ENDPOINT", - источник данных
  "method": "GET/POST/PUT/PATCH/DELETE", - метод отправки запроса
  "data": {"data1":"value1","data2":"value2"}, - данные тела запроса (необязательно)
  "target": "[ELEMENT_ID.]state[.PATH.TO.FIELD]", - адрес поля state ([] - необязательно. Данные загружаются непосредственно в поле, поверх текущих, если имеются)
  "cache": true, - параметр кэширования (необязательно, пока без реализации)
  "cacheTTL": 60000, - параметр кэширования (необязательно, пока без реализации)
},

#Макросы.

В любые данные могут интегрироваться данные через макросы {$ELEMENT_ID.state.PATH.TO.FIELD}.
Например: 
"url": "URL/TO/API/ENDPOINT/{$ELEMENT_ID.state.PATH.TO.FIELD}/{$ELEMENT_ID.state.PATH.TO.FIELD}/",
"data": {"data1":{$ELEMENT_ID.state.PATH.TO.FIELD},"data2":{$ELEMENT_ID.state.PATH.TO.FIELD}},

может быть передано всё состояние state:
"data": {$ELEMENT_ID.state}


#Серверная часть.

При старте приложения, регистрируются данные, которые поставляются на страницы в state элементов.
Конфигурация каждой страницы предусматривает раздел "dataFeed" конфигурации такой загрузки данных.
Данные могут поставляться из любых источников. 

Параметр:
"dataFeed": [
  {
    "url": "URL/TO/API/ENDPOINT",
    "method": "POST",
    "data": {"data1":"value1","data2":"value2"},
    "target": "ELEMENT_ID.state.PATH.TO.FIELD",
  },
  ...
],

При загрузке страницы, на сервере проверяется наличие запросов на странице, запрашивается API, собираются данные.
Если при сборе данных произошла ошибка, при загрузке страницы отображается сообщение (в виде Toast).


#Клиентская часть.

Во время интерактивных взаимодействий, или при загрузке страницы, в обработчике готовности страницы, 
могут быть отправлены команды на запрос данных.

Если при сборе данных произошла ошибка, отображается сообщение (в виде Toast).

"events": [
    {
      "type": "onLoad",
      "commands": [
        {
          "type": "sendRequest",
          "params": {
            "url": "URL/TO/API/ENDPOINT",
            "method": "GET",
            "target": "ELEMENT_ID.state.PATH.TO.FIELD",
          }
        },
        ...
      ]
    }
  ],

После загрузки данных в target state field, все элементы, которые подписаны на поля в state (через @... например, "value": "@filtersSection.state.textData.input"),
должны обновиться.


#Роутер.

Клиент не должен отправлять запросы в API напрямую. 
Это не запрещается, но рекомендуется работать через роутер - страница запрашивает сервер, который отправляет запрос к API и возвращает результат.

Для этого система реализует роутер.

Роутер доступен по адресу app/api/ROUTE.NAME
Для его конфигурирования, в корне конфигурации предусматривается набор роутов:

[
  {
    "path": "ROUTE.NAME",
    "url": "URL/TO/API/ENDPOINT",
  },
  ...
]

При запросе с клиента к адресу app/api/ROUTE.NAME роутера, роутер получает название роута, 
находит его в списке и отправляет принятые данные на "url" внешнего API.
Затем получает результат запроса и отдает его обратно в клиент.
