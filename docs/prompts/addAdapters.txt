Нужен функционал:
1. При получении из API, данные имеют свой формат.
2. Нужно предусмотреть адаптацию данных в формат, пригодный для отображения.
3. Для этого предусмотрены адаптеры. Два типа адаптеров предусматривается на данный момент:
- Словари, определяющие правила, замены названия свойств и их значения (если адаптируемая структура идентична, а отличие в названиях).
- JS-скрипты, которые преобразовывают полученные данные в данные CRM.
4. Адаптация производится только на стороне сервера - в местах обращении к внешним API.


Принцип адаптации данных.
1. В основной конфигурации предусмотрен раздел "adapters".
2. Структура: Название ID адаптера и набор его  свойств.
3.1. Адаптация при помощи замены ("type":"replace"):
3.1.1. Полученные из API данные проходят рекурсивный обход. 
3.1.2. По мере обхода, создается новый набор данных.
3.1.3. Каждое свойство ищется в списке правил замены "rules". Если свойство не находится - добавляется как исходное.
3.1.4. Если свойство есть в правилах
- имя заменяется на новое (параметр "name"). 
- значение заменяется на новое. Новое значение может быть любого типа. Новое значение может содержать один макрос $value$ или несколько, который должен быть заменен на исходное значение.
3.1.5. Адаптированные данные возвращаются для дальнейшей работы с ними.
3.2. Адаптация при помощи JS-скрипта ("type":"js"):
3.2.1. В поле "script" указывается адрес расположения файла скрипта на сервере.
3.2.2. Скрипт загружается в функцию, которая навход получает исходные данные, а возвращает адаптированные.
3.2.3. Адаптированные данные возвращаются для дальнейшей работы с ними.


ID адаптеров указывается в конфигурациях запросов:
- в "apiRoutes" - для роутов
{
  "path": "get-dynamic-data",
  "url": "{$config.baseURL}/mocks/api/get-dynamic-data.json",
  "adapter": "stats.analytics.js"
}

- в "dataFeed" - для фидов страниц
{
  "url": "{$config.baseURL}/mocks/api/stats/analytics.json",
  "method": "GET",
  "target": "state",
  "adapter": "stats.analytics.replace"
}


Пример конфигурации адаптеров разного типа в основной конфигурации:
"adapters": {
  "stats.analytics.replace": {
    "type":"replace",
    "rules": {
      "title":{
        "name": "label"      
      },
      "url":{
        "name": "command",
        "value": {
          "type": "download",
          "params": {
            "url": "$value$"
          }
        }
      }
    }
  },
  "stats.analytics.js": {
    "type":"js",
    "script": "adapters/analytics2menu.js"
  }
}
